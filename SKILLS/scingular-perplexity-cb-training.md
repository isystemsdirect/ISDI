# Training Perplexity on How to Use CB

## Description
This skill will teach Perplexity how to execute and manage Consolidated Batches (CBs). Perplexity will understand the CB structure, workflow, and feedback mechanisms that ensure proper execution. This training will enhance Perplexity’s ability to engage in and process multi-task scenarios, ensuring thorough execution and alignment with development processes.

## Instructions
1. **Understanding CB Structure**:
   - **CB Header**: Explains the importance of each section: Authority Chain, CB Type, Creation Date, Version, Title, and Purpose.
   - **CB Body**: Details on task inputs, process flows, and detailed instructions for each sub-task.
   - **CB Footer**: Provides instructions for return feedback, exit points, and next steps for further CBs.
      
2. **CB Execution Steps**:
   - Step 1: **CB Creation** – Explain how Perplexity needs to create a CB header with necessary metadata (e.g., Director, CDG, Receiver, etc.).
   - Step 2: **Input Data Management** – Teach Perplexity how to gather and input data accurately into the CB body. Explain how each field (such as task description, status, and report details) should be filled out.
   - Step 3: **Execution Mode** – Describe how STRICT mode must be adhered to, with high precision in data collection and feedback integration.

3. **CB Feedback & Return Process**:
   - Once the CB has been executed, explain how Perplexity needs to collect feedback loops and generate Return/Confirmation CBs.
   - Focus on how status updates, completion confirmations, and any follow-up actions should be communicated back to the CB initiator in the proper CB structure.
   - Discuss how Return CBs should mirror the CB header and footer but with updated execution statuses and outcomes.

4. **Training Through Examples**:
   - Example 1: **Task Consolidation** – Training Perplexity to execute a multi-step task, such as NVIDIA Inception Strategy for SCINGULAR.
   - Example 2: **Scenario Planning** – Teach Perplexity how to evaluate market scenarios and produce a Return CB with feedback from scenario simulations.
   - Example 3: **Collaborative Task Management** – Show how CBs can be used to coordinate multi-user inspections, producing a Return CB with updates from every contributor.

5. **Continuous CB Calibration**:
   - Introduce the 2-Phase Commit System for irreversible actions, ensuring that Perplexity can correctly evaluate and confirm irreversible actions in the CB structure.
   - Ensure CB clarity and completeness during review phases by validating each step before moving on to the next.

6. **Review and Reporting**:
   - Perplexity will create a comprehensive CB report summarizing the tasks performed, actions taken, and feedback received.
   - Review any discrepancies or exceptions in the CB process and work towards optimization.

## Output Format
- **Example CB**: A complete CB with header, body, and footer, showing the step-by-step execution of a task (e.g., AI-driven property inspection).
- **Return CB**: Confirmation from Perplexity showing the correct return and feedback from the task.

## Notes
Ensure Perplexity follows the correct CB conventions with appropriate error handling and multi-phase task tracking. This skill will empower Perplexity to handle both execution and feedback across the SCINGULAR ecosystem with precision.
