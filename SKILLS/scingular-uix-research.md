# Deep UIX R&D Skill

## Description
This skill generates detailed UI/UX research and design guidelines for SCINGULAR and OVERSCITE, leveraging UG UIX standards. It focuses on user-centric design, real-time interaction, touch scaling, and multidimensional display behavior. The output ensures all UI/UX features are governed, user-intuitive, and aligned with high-fidelity design protocols.

## Instructions
1. **UIX Governance Review**: Define Ultra-Grade UIX protocols for all interfaces in SCINGULAR and OVERSCITE.
2. **Panel Prioritization**: Ensure prime panels retain priority over auxiliary panels and handle OverHUD expansion smoothly.
3. **Display Scaling Protocol**: Create methods for viewport scaling without altering authored typography or document layout.
4. **Touch-Ready Multi-Display Setup**: Integrate multi-display readiness in OverFLIGHT™ and Drone Vision interfaces for real-time interaction.
5. **User Research**: Build user-feedback loops for iterative design validation and constant UIX refinement.
6. **Testing**: Develop UIX test cases that involve live-flight simulations, OverHUD transitions, and multi-window behaviors.
7. **Output**: Generate UI/UX design specifications, mockups, governed design assets, and testing protocols for SCINGULAR and OVERSCITE.

## Output Format
- **UI/UX Design**: E.g., Real-time control for OverFLIGHT™ with non-scroll sovereign panels, map integration, and live project branch capture.
- **Governance Guidelines**: E.g., No OverHUD should ever obscure prime flight surfaces, map should remain active, side panels must yield to OverHUD expansion.
- **Testing Protocols**: Ensure multi-display and touch interactions are validated for use in field inspections.

## Notes
Ensure governed behavior (e.g., no black slabs for panels, correct display scaling without modifying document integrity). The output must be compliant with BANE governance and suitable for field testing and deployment.
